Now that we know what data is available (`event`, `action`, and `user`), let's talk about how you
can use that data when configuring actions within a workflow.

If you're writing custom actions and want to access this data, [read our documentation on
actions](/docs/actions).

## Templating overview

Firstly, it's important to understand that actions are reusable.  For example, we provide a built-in
"HTTP" action that makes (you guessed it) HTTP requests.  You can configure this action when you use
it in a workflow, changing the URL, body, and headers that are sent each time you use the action.

When you want to use data within an action's configuration, you must use **templating** to reference
data.

Here's an example:

```
Email address: "{{ event.data.email }}"
```

This replaces the text in quotes, using the `email` field in event data.  If this workflow runs with
the following event:

```json twoslash
{
  "data": {
    "email": "whyhi@example.com"
  },
  "name": "test.event"
}
```

Then the above template will produce:

```
Email address: "whyhi@example.com"
```

You can also process data when templating:

```
Email address: "{{ event.data.email | trim | lower }}"
```

Given `event.data.email` is "`  HELLO@example.net  `" (with spaces around the email), this will
produce "hello@example.com".  Trim removes leading and trailing spaces, and lower lower-cases the
input.

## Templating functions

### String functions (text)

`lower`: lowercase the string <br /><small>`{{ event.name | lower }}`</small><br /><small>Input: "HI", output: "hi"</small>

`upper`: uppercase the string <br /><small>`{{ event.name | upper }}`</small><br /><small>Input: "hi", output: "HI"</small>

`capitalize`: capitalize the first letter, and lowercase every other letter <br /><small>`{{ event.name | capitalize }}`</small><br /><small>Input: "HELLO there", output: "Hello there"</small>

`title`: capitalize each word<br /><small>`{{ event.name | title }}`</small><br /><small>Input: "hi there", output: "Hi There"</small>

`replace`: replace parts of the text with something else <br /><small>`{{ event.name | replace(from="Hello", to="Hola") }}`</small><br /><small>Input: "Hello there", output: "Hola there"</small>

`addslashes`: adds slashes before quotes <br /><small>`{{ event.name | addslashes }}`</small><br /><small>Input: "I'm excited!", output: "I\'m excited!"</small>

`slugify`: transforms the string into a url slug<br /><small>`{{ event.name | slugify }}`</small><br /><small>Input: "Status: in progress", output: "status-in-progress"</small>

`trim`: remove leading and trailing whitespace<br /><small>`{{ event.name | trim }}`</small><br /><small>Input: " hi ", output: "hi"</small>

`trim_start`: remove leading whitespace<br /><small>`{{ event.name | trim }}`</small><br /><small>Input: " hi ", output: "hi "</small>

`trim_end`: remove trailing whitespace<br /><small>`{{ event.name | trim }}`</small><br /><small>Input: " hi ", output: " hi"</small>

`truncate`: truncate the string to a given length<br /><small>`{{ event.name | truncate(length=3, end="...") }}`</small><br /><small>Input: "Longer than three", output: "Lon..."</small>

`striptags`: removes HTML tags from the input<br /><small>`{{ event.name | striptags }}`</small><br /><small>Input: "<b\>hi!</b\>", output: "hi!"</small>

`escape`: escapes HTML<br /><small>`{{ event.name | urlencode }}`</small><br /><small>Input: "<b\>hi!</b\>", output: "`&lt;b&gt;hi!&lt;&#x2F;b&gt;`"</small>

`length`: returns the string length<br /><small>`{{ event.name | length }}`</small><br /><small>Input: "hi!", output: 3</small>

`reverse`: reverses the string<br /><small>`{{ event.name | reverse }}`</small><br /><small>Input: "hi!", output: "!ih"</small>

`urlencode`: percent-encodes non-url safe characters<br /><small>`{{ event.name | urlencode }}`</small><br /><small>Input: "why hi", output: "why%20hi"</small>

`json_encode`: transforms a value into a JSON string<br /><small>`{{ event.name | json_encode(pretty=false) }}`</small><br /><small>Input: "why hi", output: ""why hi""</small>



## Developers:  The templating engine

Under the hood, our templating is based off of the extremely powerful [rust templating engine
Tera](https://tera.netlify.app/).  We've made some changes to the engine such that it doesn't error
when fields are invalid (they become blank), and we've made some changes to compile the engine down
to webassembly.

This lets us use the templating engine on the backend, and also lets us use the same templating
engine in the UI to preview templates.
